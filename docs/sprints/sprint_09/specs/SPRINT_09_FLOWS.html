Authentication Flow

┌─────────────────────────────────────────────────────────────────┐ │ REGISTRATION FLOW │ └─────────────────────────────────────────────────────────────────┘ User → POST /api/auth/register { email, password, name } │ ▼ Zod Validate → email (format, max 320) + password (8-128) + name (1-100) │ ▼ Duplicate Check → SELECT FROM users WHERE email = ? │ ▼ Argon2id hash (19 MiB, t=2, p=1) → store in users table │ ▼ Generate synaptixlabs_id → unique cross-product identifier │ ▼ Send verification email (Resend) → 6-digit code, 15-min expiry │ ▼ 201 { userId, emailVerified: false } ┌─────────────────────────────────────────────────────────────────┐ │ LOGIN FLOW │ └─────────────────────────────────────────────────────────────────┘ User → POST /api/auth/login { email, password } │ ▼ Rate Limit → 10 req/min/IP, lockout after 5 failures │ ▼ Verify Argon2id → compare hash │ ▼ Email verified? → reject if not (redirect to verify page) │ ▼ Generate tokens: • Access JWT (1h) → returned in response body → stored in memory • Refresh JWT (7d) → HttpOnly Secure SameSite=Strict cookie │ ▼ Lazy SXC renewal → maybeRenewPlanTokens() if month boundary crossed │ ▼ 200 { accessToken, user: { id, email, plan, ... } } ┌─────────────────────────────────────────────────────────────────┐ │ FORGOT PASSWORD FLOW │ └─────────────────────────────────────────────────────────────────┘ User → POST /api/auth/forgot-password { email } │ ▼ Rate Limit → 3 req/hour/email │ ▼ Generate 15-min reset code → send via Resend │ ▼ 200 (always — never reveal if email exists) User → POST /api/auth/reset-password { code, newPassword } │ ▼ Validate code + expiry │ ▼ Argon2id hash new password → update user │ ▼ Revoke ALL refresh tokens (force re-login everywhere) │ ▼ 200 { message: 'Password reset. Please log in.' }

SXC Credit System + Paddle Billing

┌─────────────────────────────────────────────────────────────────┐ │ TOKEN CONSUMPTION MODEL │ └─────────────────────────────────────────────────────────────────┘ User Balance = plan_tokens (monthly, resets) + purchased_tokens (never expires) consumeToken(userId, amount, actionType): 1. Check plan_tokens >= amount → deduct from plan first 2. If plan_tokens < amount → deduct remainder from purchased_tokens 3. If both insufficient → return { success: false, reason: 'insufficient_balance' } 4. Log to TokenTransaction (audit trail) 5. Return { success: true, remaining: { plan, purchased } } ┌─────────────────────────────────────────────────────────────────┐ │ PADDLE WEBHOOK FLOW │ └─────────────────────────────────────────────────────────────────┘ Paddle → POST /api/webhooks/paddle { event, data } │ ▼ HMAC-SHA256 Signature Verify → reject if invalid │ ├── subscription.created → Update plan + plan_tokens + status ├── subscription.updated → Update plan tier + billing period ├── subscription.canceled → Mark canceled, allow until period end ├── subscription.past_due → Flag payment issue └── transaction.completed → Add purchased_tokens (token pack)

Security Architecture

Authentication
Argon2id (OWASP 2025), JWT (15-min access + 7d refresh rotation), HttpOnly cookies + fingerprint
Paddle Webhooks
HMAC-SHA256 signature verification on every event
Input Validation
Zod schemas on every API endpoint, parameterized SQL
Rate Limiting
10 req/min/IP on auth, 5 failures → 15 min lockout
CSRF Protection
SameSite=Strict cookies + double-submit token
Security Headers
Helmet.js: CSP, HSTS, X-Frame-Options, nosniff
PII Protection
Scrub PII from screenshots before AI analysis
Token Storage
Access token in memory (never localStorage). Refresh in HttpOnly cookie.
GOD Admin
admin@synaptixlabs.ai — immutable, cannot be deleted/downgraded
